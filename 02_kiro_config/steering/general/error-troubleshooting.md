# Error Troubleshooting

## Purpose
This rule ensures systematic debugging and troubleshooting when errors are encountered.

## Instructions
- MUST debug and troubleshoot systematically when any error is encountered, including analyzing error messages, checking logs, and identifying root causes before attempting fixes. (ID: DEBUG_ERRORS)
- MUST use available MCP tools for identifying fixes, solutions, and best practices when troubleshooting errors. (ID: MCP_ERROR_FIXES)

## Priority
High

## Error Handling
- If MCP tools are not available for error resolution, use available debugging tools and document the limitation
- If error cannot be resolved, document the error details and attempted solutions
